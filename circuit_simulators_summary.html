üé® Analog IC Design Flow

The analog IC design process involves several critical steps from concept to fabrication:

Step 1: Draw Schematic

Use a schematic editor to draw the circuit diagram with all components and connections.

Step 2: Create Netlist

A code that describes the circuit in the simulator language. The netlist contains all circuit connections and component values.

Step 3: Run Circuit Simulator

Use SPICE to simulate the circuit behavior. Models describe the behavior of each component.

Step 4: Observe and Process Results

Analyze simulation outputs, examine effects of non-idealities and environment effects.

Step 5: Modify and Iterate

Adjust the circuit based on results and repeat until satisfied with performance.

Step 6: Draw Layout

Create the actual physical layout of transistors, resistors, capacitors, metal interconnects, PADs, ESD diodes, and power lines. Be careful with coupling!

Step 7: Design Rule Check (DRC)

Verify that the layout respects design rules dictated by the foundry (spacing between diffusions, minimum metal line width, etc.).

Step 8: Layout vs Schematic (LVS)

Create a netlist from the layout and ensure it matches the schematic netlist. They must be the same circuit!

Step 9: Extract Parasitics

Add unintentional parasitic capacitances and resistances created in the layout to the netlist.

Step 10: Simulate with Parasitics

Re-run simulations with extracted parasitics and fix any degradation that may have resulted.

Step 11: Send for Fabrication

Submit the final design to the foundry and pray! üôè

EDA Vendors: Cadence, Mentor Graphics, Synopsys, Tanner EDA, Multisim, and others provide Electronic Design Automation tools.

üî¨ SPICE Simulator

What is SPICE?

SPICE = Simulation Program with Integrated Circuit Emphasis

A general-purpose circuit simulation program for nonlinear DC, nonlinear transient, and linear AC analyses. Developed in the 1970s by Berkeley.

Supported Components:

  • Resistors, capacitors, inductors, mutual inductors
  • Independent voltage and current sources
  • Several types of dependent sources
  • Lossless and lossy transmission lines
  • Switches, uniform distributed RC lines
  • Common semiconductor devices: diode, BJT, FET

How SPICE Works:

  • Creates all equations describing the circuit (KCL, KVL, device equations)
  • Solves all equations:
    • Linear equations ‚Üí Easy! Invert the matrix
    • Non-linear equations (active elements) ‚Üí Solve iteratively using Newton-Raphson Method
Convergence Issues: Convergence is a notorious problem with simulators, especially for circuits with non-linear elements.

üìä SPICE MOS Models

SPICE uses a variety of transistor models with different trade-offs between complexity and accuracy:

Model Levels:

  • Level 1: Simple model
  • Level 2: Simple model
  • Level 3: Simple model
  • BSIM 1: Berkeley Short-Channel IGFET Model
  • BSIM 2: Very elaborate model
  • BSIM 3: Very elaborate model
  • BSIM 4: Current standard, most elaborate model

BSIM-4 Model Features:

1. Body Effect

Threshold voltage is modulated by the source-to-body voltage. Also includes dependence of VTH on length and width.

2. Velocity Saturation (Short Channel Effect)

When the channel is short, lateral electric field is high and electrons cannot reach theoretical velocity. Current is not as high as basic equations suggest.

5. Drain-Induced Barrier Lowering (DIBL)

  • Long-channel: Depletion region from drain is far from gate/source interaction, no impact on channel creation
  • Short-channel: Drain depletion region affects fields and charge distribution at gate/source. Higher drain voltage reduces threshold voltage (drain helps create the channel)

6. Subthreshold Conduction

Current flow below threshold voltage shows exponential relationship with VGS.

7. Gate Oxide Tunneling

As technology nodes advance, oxide thickness gets thinner (1nm or less). Some electrons tunnel through oxide creating gate current (Fowler-Nordheim tunneling).

8. Additional Features:

  • Continuous and differentiable I-V characteristics for good convergence
  • Enhanced models for diffusion capacitance and resistance
  • More elaborate modeling of gate capacitances

üîç Types of SPICE Analysis

1. DC Analysis

Standard DC operating point analysis - similar to hand calculations.

2. AC Small-Signal Analysis

  • Find DC solution first
  • Find linearized, small-signal gain from all inputs to output as a function of frequency
  • This is a small-signal analysis (linearized)

3. Transient Analysis

  • Actual large-signal analysis of the circuit (no linearization)
  • Circuit is solved at every timestep
  • Time-consuming: need to simulate long enough to see complete period(s), yet timestep must be ~10-100 times smaller than smallest time period

4. Pole-Zero Analysis

Determines poles and zeros of transfer functions.

5. Small-Signal Distortion Analysis

Analyzes harmonic distortion in small-signal regime.

6. Sensitivity Analysis

Determines how circuit performance changes with component variations.

7. Noise Analysis

AC analysis with noise added to sources and elements (linearized).

8. PSS, PAC, Pnoise (Periodic Steady State)

Problem: What if stimulus consists of large signal plus small signal?

  • Example 1: Mixer with large LO signal and small RF signal
  • Example 2: Oscillator with large oscillation but small noise signals
  • Can't use AC analysis (due to large signal) or transient (too slow, doesn't account for noise)
  • Solution: PSS solves steady-state response over fundamental frequency, then superimposes small signal on it

üíª Digital IC Design Flow

Digital design is fundamentally different from analog design - it uses a top-down approach rather than bottom-up.

Key Concept: Digital circuits are made of well-known blocks (gates). No need for custom transistor-level design. Instead, build the system from the top down!

Main Steps:

1. Design Entry (RTL)

Write Hardware Description Language (VHDL or Verilog) code describing signal flow from register to register.

2. Simulation/Verification

EDA tools simulate RTL and verify functionality (e.g., Conformal from Cadence). Also estimate power consumption (e.g., Joules from Cadence).

3. Synthesis

Convert RTL into gates (NANDs, NORs, Flip-Flops, etc.) using standard cells from technology library. Tool automatically optimizes logic (e.g., Genus from Cadence).

  • Timing verification (e.g., Tempus from Cadence)
  • Power integrity check (e.g., Voltus from Cadence)
4. Automatic Place & Route (APR)

Each gate is replaced by its standard cell (pre-laid-out transistors). Tool automatically places cells, routes metal interconnects and power rails, and optimizes clock tree (e.g., Innovus from Cadence). Many iterations needed to meet timing.

5. Gate-Level Simulations

More accurate simulations on gate-level netlist (e.g., Xcelium from Cadence). Captures effects missed in RTL:

  • More accurate delay simulations
  • Interaction of multiple voltage domains
  • Glitches caused by slow clock edges
  • Power-on-reset sequence verification
6. Tape-Out

Final design sent for fabrication.

üìù RTL (Register Transfer Level)

Using Hardware Description Languages (HDL) like VHDL or Verilog, write code describing exact signal flow from one register to the next.

Example: Simple AND Gate in VHDL

library ieee; use ieee.std_logic_1164.all; entity example_and is port ( input_1 : in std_logic; input_2 : in std_logic; and_result : out std_logic ); end example_and; architecture rtl of example_and is signal and_gate : std_logic; begin and_gate <= input_1 and input_2; and_result <= and_gate; end rtl;
Scalability: Very complex systems can be written in RTL hierarchically, making it possible to design million-gate designs efficiently.
